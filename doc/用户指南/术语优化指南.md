# DeeChat 用户友好术语指南

## 🎯 设计理念
DeeChat 基于用户视角设计，避免技术术语，强调实用性和易理解性。

## 📚 术语映射表

### 🔄 核心概念替换

| 原技术术语 | DeeChat用户友好术语 | 说明 |
|----------|------------------|------|
| **Workspace** | **对话空间** | 用户理解为不同的聊天环境/场景 |
| **Thread** | **对话记录** | 直观的历史对话概念 |
| **Vector Database** | **知识库** | 用户的知识存储和检索系统 |
| **Embedding** | **文档索引** | 文档被处理成可搜索的格式 |
| **Vector Search** | **智能搜索** | 基于语义理解的搜索功能 |
| **Chunking** | **内容分段** | 文档被智能分段以便处理 |
| **LLM Provider** | **AI模型** | 用户直接理解的AI模型概念 |
| **System Prompt** | **AI助手设定** | AI角色的配置和指令 |
| **Token Limit** | **对话长度限制** | 用户理解的对话长度概念 |

### 💡 用户体验原则

1. **功能导向**：强调"能做什么"而不是"技术实现"
2. **场景化语言**：使用用户熟悉的场景词汇
3. **一致性**：整个应用保持术语一致
4. **直观性**：用户一看就明白功能用途

### 🎨 视觉配合

- **图标选择**：使用直观的图标（📚知识库、💬对话、🔍搜索）
- **颜色含义**：绿色表示成功、蓝色表示信息、红色表示警告
- **动画反馈**：操作提供清晰的视觉反馈

## 🚀 实施步骤

1. **翻译文件更新**：修改所有语言包的术语
2. **界面文案修改**：更新按钮、标题、提示文案
3. **帮助文档重写**：用用户友好语言重写使用指南
4. **测试验证**：确保用户体验流畅自然
